# Workflow Migration Brief

A team is migrating from ad hoc manual work to a verified agent-assisted workflow. The source material compares the old operating mode with the new one for the same recurring task.

Before:

- Requests arrive in chat or email with inconsistent context.
- Operators manually collect files, retype details, and improvise tool usage.
- Verification is optional, so output quality depends on memory and luck.
- Decisions and follow-up steps are easy to lose.

After:

- A reusable intake pattern captures scope, risks, and artifacts up front.
- Agent steps are explicit, tool-backed, and verified before handoff.
- High-risk side effects are reviewed before execution.
- Outputs include evidence, changed files, and known limits.

The diagram should highlight the operating-mode shift, not every sub-step in the workflow.
